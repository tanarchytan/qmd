## Rollback recipe (if publish fails catastrophically)

1. `npm unpublish @tanarchy/lotl@1.0.0 --force` — **only within 72h** of publish
2. Delete the v1.0.0 tag: `git push origin --delete v1.0.0`
3. Delete GitHub release: `gh release delete v1.0.0`
4. Fix root cause
5. Retag + push to retry

## What to do if Phase 5 (KG + fact-aug) comes up before v1.0.0 ships

Phase 5 is a structural change (schema migration). Cleanest path:
- Ship v1.0.0 WITHOUT Phase 5
- Land Phase 5 as v1.1.0-beta.1 after
- Use the standard release flow again

Don't try to ship Phase 5 as part of v1.0.0 — the extra ingest time + LLM
cost delays release past the clean-cutoff.
